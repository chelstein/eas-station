Design Principles

Guidelines for consistent design

Consistency
  • Every page MUST use the standard page header
  • Use section headers to divide content
  • All stat cards use gradient backgrounds
  • Icons always precede text with me-1 or me-2 spacing
  • Always use the defined grid classes for layouts
Visual Hierarchy
  • Page header → Section headers → Cards
  • Important actions use solid buttons (primary, success)
  • Secondary actions use outline buttons
  • Group related items in cards
  • Use info boxes for contextual help
Spacing
  • Page header: mb-4 (margin-bottom: 2rem)
  • Section spacing: mb-4 or mb-5
  • Card spacing: mb-3 or mb-4
  • Button groups: gap-2
  • Form fields: g-3 in rows
Accessibility
  • Always include icon classes alongside text
  • Use semantic HTML (h1, h2, section, etc.)
  • Forms have visible labels
  • Buttons have descriptive text or aria-labels
  • Color is not the only indicator (use icons too)
