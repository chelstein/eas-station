Application Settings

Configure logging, file storage, and other application-level settings

Logging
Controls how much detail is logged. DEBUG is very verbose; INFO is recommended for production. Changes apply immediately without a restart.
Path to the application log file (relative to the project root, or an absolute path). A service restart is required for log file path changes to take effect.
File Storage
Absolute path where uploaded files are stored. The directory will be created automatically if it does not exist. A service restart is required for this change to take effect.
Password Policy
Minimum number of characters required. Must be between 1 and 128.
These rules are enforced when creating or resetting administrator passwords. Existing passwords are not affected until they are changed.
days
Number of days before passwords expire. Set to 0 to disable expiration. Admins will receive a warning banner within 14 days of their password expiry date.
Bootstrap Settings

The following settings must still be configured via environment variables in your .env file because they are required before the database connection is available:

  • SECRET_KEY — Flask session security key
  • DATABASE_URL — PostgreSQL connection URL
  • CACHE_REDIS_URL — Redis connection URL
  • CACHE_TYPE — Cache backend type
  • FLASK_ENV / FLASK_DEBUG — Flask startup mode

Help
Log Levels
  • DEBUG — All messages including verbose diagnostics
  • INFO — Normal operational messages (recommended)
  • WARNING — Only warnings and errors
  • ERROR — Only errors
  • CRITICAL — Only critical failures

Log level changes take effect immediately. Other settings require a service restart.
