Install Postfix

Postfix is a standard Linux mail transfer agent. Installing it here adds it to the system via apt-get and configures it as a local outbound-only relay. No email will be received — only outbound alert emails are sent.

How It Works
  1. Install Postfix on this server.
  2. Set the hostname — must match your server's reverse-DNS record in Vultr.
  3. Apply & Restart — EAS Station writes the Postfix config and starts the service.
  4. Use Postfix — one click sets the notification SMTP URL to smtp://localhost:25.
  5. Send a test to verify delivery before going live.
Vultr Setup

Vultr blocks port 25 by default. To send mail directly:

  1. Log into my.vultr.com
  2. Open a Support ticket requesting SMTP port 25 unblocked for your instance IP
  3. Set a Reverse DNS (PTR) record for your IP under Products → Instances → your instance → Settings → IPv4
  4. Add an SPF TXT record to your domain's DNS: v=spf1 ip4:YOUR_IP ~all

Vultr typically responds to SMTP unblock requests within a few hours for accounts in good standing.

Deliverability Tips
  • PTR record = hostname in config above
  • SPF record covers your server's IP
  • Send-only server — no inbound mail configured
  • Postfix enforces opportunistic TLS outbound
  • If mail goes to spam: add DKIM signing via opendkim (advanced — coming soon)
