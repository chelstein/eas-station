How County Boundaries Work
What is this table for?

IPAWS/NWS alerts often carry SAME geocodes (6-digit codes like 039137) instead of a polygon. EAS Station converts those codes into highlighted county outlines on the alert detail map by looking up each code in us_county_boundaries. Without this table, those alerts show "Alert has no geographic data."

SAME code format

A 6-digit code: 0 SS CCC where SS is the 2-digit state FIPS and CCC the 3-digit county FIPS.

SAME CodeState FIPSCounty FIPSCensus GEOIDCounty
0391373913739137Putnam, OH
0010010100101001Autauga, AL
0060370603706037Los Angeles, CA
03900039000–All of Ohio (statewide)
How to load the data
  1. Automatic (recommended): EAS Station auto-loads the bundled Census shapefile on first startup if the table is empty. {% if bundled_shapefile_available %}
    Bundled file found at {{ bundled_shapefile_path }} {% else %}
    Bundled file not found — use the Upload option below or re-run update.sh. {% endif %}
  2. Manual (admin UI): Click Load Counties in the "Load County Boundaries" section above. You can filter to one state to load incrementally.
  3. Command line: 
    cd /opt/eas-station
source venv/bin/activate
python scripts/load_us_county_boundaries.py
  4. Download fresh shapefile from the Census Bureau (if the bundled file is out of date):
    census.gov → Cartographic Boundary Files → Counties — download the 500k ZIP, then use Upload Custom Shapefile above.
Verify via API

Check table status programmatically: 

GET /admin/county_boundaries/status
GET /admin/county_boundaries/lookup?same=039137,001001
GET /admin/county_boundaries/geojson?state=OH
