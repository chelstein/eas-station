# Archive Directory

This directory contains historical documentation and fix documents that are no longer actively maintained but preserved for reference.

## Historical Fixes

The `historical-fixes/` subdirectory contains documentation of specific bugs and fixes from the development history of EAS Station. These documents provide context for past issues and their resolutions.

**Note**: For current troubleshooting information, see the [docs/troubleshooting/](../troubleshooting/) directory.

### Contents

- **STARTUP_FIX_SUMMARY.md** - Web app startup issue fix (v2.27.4)
- **DEPLOYMENT_INSTRUCTIONS.md** - Website timeout fix deployment (v2.27.5)
- **FIXES_APPLIED.md** - Alert feeds configuration fix
- **WEBSITE_504_FIX.md** - Website 504 error fix (PR #1356)
- **FIX_FOR_TIMEOUT_ISSUE.md** - wsgi.py initialization fix
- **DEPLOYED_SYSTEM_FIX.md** - Virtual environment package conflicts fix
- **QUICK_FIX.md** - One-command deployment fix
- **QUICK_DATABASE_FIX.md** - Database authentication troubleshooting
- **ROOT_CAUSE_ANALYSIS.md** - PR #1356 breaking changes analysis
- **ISO_BUILD_READY.md** - Repository cleanup for ISO build (v2.19.4)
- **PATH_AUDIT.md** - Hardcoded path audit and fixes

## Purpose

These documents are preserved to:
- Provide historical context for code changes
- Document lessons learned during development
- Help understand why certain architectural decisions were made
- Serve as reference for similar future issues

For current documentation, always refer to the main documentation directories rather than these archived files.
